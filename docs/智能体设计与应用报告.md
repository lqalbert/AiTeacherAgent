# 智能课堂助教（AiTeacherAgent）设计与应用报告

> **课例类型**：人工智能教学创新应用 · AI 教育智能体设计与创新应用  
> **智能体名称**：智能课堂助教（AiTeacherAgent）  
> **适用学科**：中小学各学科课堂（本课例以英语语法教学为典型场景）

---

## 一、智能体开发背景

### 1.1 政策与教学背景

《教育信息化 2.0 行动计划》《关于加强新时代教育体系建设的意见》等文件强调：推动人工智能与教育教学深度融合，减轻教师重复性劳动，支持精准教研与课后复盘。传统课堂中，教师需要同时完成「讲授—板书/课件—互动—记录—反思」多项任务，**注意力被分散**，课后整理笔记、分析重难点、出题往往耗时且难以还原课堂真实过程。

### 1.2 技术背景

大语言模型（LLM）与实时语音大模型（ASR-LLM）已具备较强的语义理解与学科文本生成能力；Web 端可实现课件放映、麦克风采集与实时通信。将二者结合，可构建**「感知—记忆—推理—行动」闭环**的课堂智能体，而非单一功能的工具软件。

### 1.3 开发动机

本智能体面向一线教师真实授课场景设计，目标是在**不改变既有教学习惯**（仍用 PPT、仍口头讲授）的前提下，由智能体自动完成听写、记录、分析与评价，形成可沉淀、可导出、可复盘的数字课例资产。

## 二、拟解决的核心问题

| 序号 | 教学痛点 | 智能体解决思路 |
|------|----------|----------------|
| 1 | 课堂口述内容难以完整留存 | 实时语音转写 + AI 术语校对，生成结构化字幕 |
| 2 | 课件与讲解不同步，课后难复盘 | 自动记录翻页时间轴，支持字幕与 PPT 同步回放 |
| 3 | 课后整理重难点、写总结耗时长 | 结束本节后 AI 自动分析重点、难点、知识点 |
| 4 | 缺乏对本节课教学过程的客观评价 | AI 生成课堂评价（综合分、分项分、亮点与改进建议） |
| 5 | 习题与知识结构需教师二次加工 | AI 自动生成思维导图与分层课后习题 |
| 6 | 同一课程多节课次难以管理 | 多课次（Round）记忆与独立报告 |

## 三、智能体定位与架构

### 3.1 智能体定义（答辩用语）

**智能课堂助教**是一个面向 K12/中学课堂的 **AI 教育智能体**，具备：

- **感知（Perception）**：麦克风语音流、PPT 翻页事件、课次状态  
- **记忆（Memory）**：课程元数据、按节转写、翻页时间轴、历史分析报告  
- **推理（Reasoning）**：多模块 LLM 协同（字幕校对、教研分析、课堂评价、思维导图、出题）  
- **行动（Action）**：字幕展示、报告生成、课件回放、Word/Markdown 导出  
- **目标（Goal）**：完成「讲授—记录—分析—评价—复盘」全链路闭环  

### 3.2 系统架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    教师（用户）                               │
│         上传课件 · 讲授 · 翻页 · 结束本节                    │
└───────────────────────────┬─────────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────────┐
│              智能课堂助教 Agent（编排层）                    │
│  课次管理 │ 工作流触发 │ 上下文组装 │ 多 AI 模块调度         │
└─┬─────────┬─────────┬─────────┬─────────┬───────────────────┘
  │         │         │         │         │
  ▼         ▼         ▼         ▼         ▼
听写技能  记录技能  教研技能  评价技能  复盘技能
(ASR+校对)(DB记忆) (分析+导图)(课堂评价)(回放+导出)
  │         │         │         │         │
  └─────────┴────┬────┴─────────┴─────────┘
                 ▼
        SQLite 课堂记忆库 + 课件文件
```

### 3.3 与「普通软件」的区别（创新性表述）

| 维度 | 普通教学软件 | 本智能体 |
|------|-------------|----------|
| 交互 | 固定按钮与表单 | 按课次工作流自主串联多能力 |
| 输出 | 单一转写或单一报告 | 转写→分析→评价→导图→习题 全 AI 生成 |
| 上下文 | 无状态 | 按节记忆转写、翻页、报告 |
| 增强 | 规则处理 | LLM 结合学科热词与课堂上下文校对 |

## 四、核心功能（智能体技能清单）

### 技能 1：授课技能
- 新建/继续课程，上传 `.pptx`，浏览器内放映与全屏
- 支持 PPT 动画切换、键盘翻页

### 技能 2：听写技能（感知 + 推理）
- 讯飞实时语音转写大模型（`rtasr_llm`，教育场景 `pd=edu`）
- DeepSeek/通义千问 **AI 字幕校对**（学科术语、同音错字）
- 可选学科热词 `ASR_HOTWORDS` 提升专有名词准确率

### 技能 3：记录技能（记忆）
- 按课次保存转写片段（`transcript_segments`）
- 按课次保存翻页事件（`slide_events`）
- 多节课次独立存储，支持删除与继续上课

### 技能 4：教研技能（推理 + 行动）
- AI 课堂总结、重点、难点、知识点标签
- AI 思维导图（知识结构）
- AI 课后习题（选择/填空/简答，难度与课堂匹配）

### 技能 5：评价技能（推理 + 行动）
- AI 课堂综合评价（200–400 字）
- 教学综合评分 + 分项评分（内容/逻辑/重点/表达）
- 教学亮点与改进建议

### 技能 6：复盘技能（行动）
- 课件回放：字幕与 PPT 时间轴同步
- 报告查看与 Word/Markdown 导出

## 五、智能体核心搭建步骤（重点：可截图、可演示）

> 以下步骤对应本项目**真实代码路径**，可作为报告「搭建过程」章节，并配合界面截图与关键代码片段。

### 步骤 1：定义智能体目标与工作流

明确智能体要完成的标准剧本：

```
建课 → 开始转写 → 讲授（自动记录翻页+字幕）→ 结束本节
     → 触发 AI 全量分析 → 查看报告/回放 → 导出
```

**对应实现**：`client/src/pages/Home.tsx`、`Classroom.tsx`、`Report.tsx` 三页闭环。

---

### 步骤 2：构建感知层（多模态输入）

| 感知源 | 技术实现 | 代码位置 |
|--------|----------|----------|
| 语音 | WebSocket 音频流 → 讯飞 ASR | `client/hooks/useAudioCapture.ts` → `server/asr/bridge.js` |
| 翻页 | 前端 slide 变化 → HTTP 记录 | `Classroom.tsx` → `POST /api/sessions/:id/slide` |
| 课次状态 | Session / Round 状态机 | `server/db/store.js` |

**答辩要点**：智能体不仅「听」，还「看」课件进度，形成**语音—课件对齐**的课堂数字孪生。

---

### 步骤 3：构建记忆层（结构化课堂记忆）

设计 SQLite 数据模型，按课次隔离：

```
sessions（课程）
  └── lesson_rounds（第几节课）
        ├── transcript_segments（转写记忆）
        ├── slide_events（翻页记忆）
        ├── analysis_results（教研+评价记忆）
        └── generated_questions（习题记忆）
```

**对应实现**：`server/db/schema.sql`、`server/db/store.js`

**答辩要点**：这是智能体的**长期记忆**，支持多节课、课后追问与复盘。

---

### 步骤 4：构建推理层（多 AI 子模块协同）

采用「专能模块」设计，各模块独立 Prompt，由编排层顺序调用：

| 模块 | 功能 | 代码 |
|------|------|------|
| 字幕校对 | 实时修正 ASR 误识别 | `server/ai/transcriptPolish.js` |
| 教研分析 | 总结/重难点/知识点 | `server/ai/analyze.js` → `analyzeLesson` |
| 课堂评价 | 评分+亮点+建议 | `server/ai/analyze.js` → `generateLessonEvaluation` |
| 思维导图 | 知识结构树 | `server/ai/mindMap.js` |
| 习题生成 | 分层出题 | `server/ai/analyze.js` → `generateQuiz` |

统一 LLM 网关：`server/ai/chat.js`（DeepSeek / DashScope 可切换）

**编排入口**：`runFullAnalysis()` — 结束本节或手动「生成报告」时自动执行全流程。

---

### 步骤 5：构建行动层（工具 API）

将智能体能力封装为可调用工具（REST + WebSocket）：

| 工具 | 接口 |
|------|------|
| 建课 | `POST /api/sessions` |
| 继续上课 | `POST /api/sessions/:id/continue` |
| 实时转写 | `ws://host/ws/asr?sessionId=` |
| 记录翻页 | `POST /api/sessions/:id/slide` |
| 结束本节+触发分析 | `POST /api/sessions/:id/end` |
| 生成/重新分析 | `POST /api/sessions/:id/analyze` |
| 获取报告 | `GET /api/sessions/:id/report?round=` |
| 导出 | `GET /api/sessions/:id/export?format=docx\|md` |

**对应实现**：`server/index.js`

---

### 步骤 6：构建交互层（智能体界面）

| 页面 | 智能体角色 |
|------|-----------|
| 首页 | 任务列表 + 启动/继续授课 |
| 课堂页 | 感知展示（PPT+实时字幕）+ 控制（转写/结束） |
| 报告页 | 推理结果展示（评价/总结/导图/习题） |
| 回放模式 | 复盘（字幕与课件同步） |

**对应实现**：`client/src/pages/*.tsx`

---

### 步骤 7：联调与部署

```bash
npm run setup
cp .env.example .env   # 配置讯飞 + DeepSeek/DashScope
npm run dev            # 开发：http://localhost:5173
npm run build && npm start  # 生产部署
```

环境变量即智能体「能力开关」：ASR 提供商、AI 提供商、学科热词、AI 校对开关等。

---

### 步骤 8：验证智能体闭环（演示脚本）

课例演示建议按以下脚本录制（3–5 分钟）：

1. **建课**：上传一节英语语法 PPT  
2. **授课**：开始转写，讲 1–2 分钟，翻 2–3 页（展示 AI 校对标签）  
3. **结束**：点击「结束本节」，自动跳转报告页  
4. **成效**：展示 AI 生成的——课堂评价、总结、思维导图、习题  
5. **复盘**：进入课件回放，拖动进度条展示字幕与 PPT 同步  
6. **导出**：下载 Word 报告

## 六、核心应用功能及效果

### 6.1 应用流程图

```
教师上传课件
    ↓
智能体进入「授课模式」（听写技能启动）
    ↓
实时字幕（ASR + AI 校对）+ 翻页自动入库
    ↓

    ↓
智能体自动执行教研流水线（全 AI 生成）
    ├── 课堂总结 / 重难点
    ├── 课堂评价（评分+建议）
    ├── 思维导图
    └── 课后习题
    ↓
教师查看报告 / 课件回放 / 导出 Word
```

| 功能 | 应用前 | 应用后（智能体） |
|------|--------|------------------|

| 术语准确 | ASR 同音错误多 | AI 结合学科热词校对 |
| 课后分析 | 30–60 分钟人工整理 | 结束本节后自动生成 |

## 七、应用成效及创新性

### 7.1 应用成效

- **减负**：课后整理与出题时间显著缩短  
- **提质**：课堂过程可回放、可分析，支持教学反思  
- **沉淀**：形成可导出的数字课例资产（Word/Markdown）  
- **适切**：不改变 PPT 授课习惯，低迁移成本  

### 7.2 创新性归纳（建议答辩用「三点」）

1. **多模态课堂记忆**：语音转写 + PPT 翻页时间轴双通道记忆，支持同步复盘，超越单一录音或单一转写工具。  
2. **「听—记—析—评」一体化智能体流水线**：不是多个 disconnected 工具，而是结束本节后自动触发的 AI 教研工作流（总结、评价、导图、习题全部由 LLM 生成）。  
3. **学科增强的实时听写**：ASR 大模型 + LLM 二次校对 + 学科热词，形成「感知—推理」增强回路，提升课堂字幕可用性。  

### 7.3 可扩展方向（展望，非本次必做）

- 增加自然语言对话入口（课堂助手 Chat）  
- 对接 MCP，供外部 Agent 调用本系统工具  
- 班级维度的学情汇总智能体

## 十、答辩常见问题预备

**Q：这是智能体还是普通 Web 应用？**  
A：本系统具备智能体核心要素：多源感知、结构化记忆、多模块 LLM 推理、工具化行动与标准工作流编排；界面是智能体的交互外壳，后端 `runFullAnalysis` 与 ASR 流水线是智能体的「大脑与神经」。

**Q：AI 生成内容如何保证与课堂一致？**  
A：所有分析 Prompt 明确要求「仅基于转写事实、禁止编造」；评价、总结、导图、习题均输入完整/按页转写作为上下文。

**Q：没有聊天框还算智能体吗？**  
A：智能体不等于聊天机器人。本智能体以**任务驱动工作流**为形态：教师按课例剧本操作，智能体在关键节点自主完成听写、分析、评价与导出，符合「AI 教育智能体」中 **Agent = 目标 + 感知 + 记忆 + 推理 + 行动** 的定义。
