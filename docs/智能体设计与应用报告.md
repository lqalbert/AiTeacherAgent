## 摘要

「智课随行」面向中职真实课堂，在教师不改变「PPT + 口头讲授」习惯的前提下，由智能体自动完成课中听写与字幕呈现、课件翻页对齐与课堂记忆沉淀，并在结束后依据**课件原文、课堂转写、知识库证据**与教师自定义**评价标准**，一键生成教学评价、重难点、思维导图与课后习题，支持同步回放与 Word 导出。

本作品定位为**任务型教育智能体**而非聊天机器人：以一节课的完整任务为目标，编排听写、对齐、评课、结构、出题等技能，贯通**备课配置—课中随行—课后评课**全链路，服务学生听课可及性与教师教研减负提质。

## 一、智能体开发背景

### 1.1 政策与教育背景

推动人工智能与教育教学深度融合，是教育数字化转型的重要方向。《教育信息化 2.0 行动计划》等文件强调以智能技术减轻教师重复性劳动、支持精准教学与反思性教研。职业教育强调理实一体、过程可评、证据可留；新课标与核心素养导向下，课堂不仅要「讲清楚知识点」，更要形成可回看、可评价、可改进的教学过程证据。

中职课堂具有鲜明特征：专业术语密集、实训演示与课件讲解交织、学生听力与注意力差异大。教师需同时完成讲授、操作课件、组织互动与课堂管理，课后仍要归纳重难点、撰写教学反思、编制巩固练习。**过程性数据高度依赖记忆与手工笔记**，难以完整还原「第几页讲了什么、难点卡在何处、评价依据是什么」。

### 1.2 技术背景

当前技术条件已支撑「课堂智能体」落地：

| 能力层 | 技术条件 | 本项目采用 |
|--------|----------|------------|
| 感知 | 实时语音大模型、浏览器音视频采集 | 讯飞实时语音转写（教育场景）+ Web 麦克风 |
| 记忆 | 结构化数据库、文件存储 | SQLite 课次记忆 + 课件/知识库文件 |
| 推理 | 大语言模型语义理解与生成 | DeepSeek / 通义千问多模块协同 |
| 行动 | Web 交互、文档导出 | 实时字幕、报告生成、同步回放、Word/Markdown 导出 |
| 编排 | 工作流与配置化 Agent | 智能体工作台 + `runFullAnalysis` 全量分析流水线 |

将上述能力按「感知—记忆—推理—行动」闭环编排，即可从「单点工具」升级为可复用的**教育智能体**。

### 1.3 开发动机

本智能体面向一线中职教师真实授课设计，目标是：

1. **不改变教学习惯**：仍用 PPT、仍口头讲授，智能体「随行」而非「替代」；
2. **课中服务学生**：实时字幕降低听课门槛，提升课堂信息可及性；
3. **课后服务教师**：自动完成总结、评价、导图、出题等教研重复劳动；
4. **证据可沉淀**：形成可导出、可回放的数字课例资产，支撑教研与成长。

## 二、拟解决的问题

| 序号 | 教学痛点 | 对应教学环节 | 智能体解决思路 |
|------|----------|--------------|----------------|
| 1 | 课堂口述内容难以完整留存，学生跟不上节奏 | 授课 | 实时 ASR + AI 术语校对，生成结构化字幕 |
| 2 | 课件与讲解不同步，课后难复盘「讲到哪一页」 | 授课 / 评价 | 自动记录翻页时间轴，字幕与 PPT 同步回放 |
| 3 | 课后整理重难点、写总结耗时长且易主观 | 评价 | 结束听课后 AI 基于证据自动分析总结/重点/难点 |
| 4 | 教学评价缺少过程证据，易空泛套话 | 评价 | 按自定义评价标准生成亮点与改进建议，要求页码/转写依据 |
| 5 | 习题与知识结构需教师二次加工 | 备课 / 评价 | AI 生成思维导图与分层课后习题（题型题量可配） |
| 6 | 评课标准、知识库因校/因专业而异，难以复用 | 备课 | 智能体工作台配置角色、标准、知识库、报告项与技能工作流 |
| 7 | 多位教师课次与配置易混用，数据边界不清 | 全流程 | 账号鉴权与按用户隔离配置、知识库、课次数据 |

**问题本质**：不是缺少「录音」或「聊天问答」，而是缺少一个能在真实课堂中**自动感知、结构化记忆、按标准推理、输出可行动成果**的教育智能体。

## 三、核心功能

### 3.1 智能体定义与五要素

**智课随行**是面向中职课堂的 **AI 教育智能体**，具备：

| 要素 | 内涵 | 本系统实现 |
|------|------|------------|
| **目标（Goal）** | 完成「配置—授课—分析—评价—复盘」闭环 | 工作台配置 → 听课 → 结束并生成报告 → 回放/导出 |
| **感知（Perception）** | 多源课堂信号 | 麦克风语音流、PPT 翻页事件、课次状态 |
| **记忆（Memory）** | 结构化课堂记忆 | 转写片段、翻页时间轴、分析结果、习题、知识库语料 |
| **推理（Reasoning）** | 多技能 LLM 协同 | 字幕校对、教研分析、评课、思维导图、出题 |
| **行动（Action）** | 可交付产出 | 实时字幕、成长报告、同步回放、Word/Markdown 导出 |

> **答辩要点**：智能体 ≠ 聊天机器人。本作品以**任务驱动工作流**为形态，教师按课例剧本操作，智能体在关键节点自主完成听写、对齐、分析与导出。

### 3.2 系统架构

```
┌──────────────────────────────────────────────────────────────┐
│                     教师（用户）                               │
│   工作台配置 · 建课上传 PPT · 讲授翻页 · 结束并生成报告         │
└────────────────────────────┬─────────────────────────────────┘
                             │
┌────────────────────────────▼─────────────────────────────────┐
│              智课随行 Agent（编排层）                          │
│  角色/标准/知识库/技能配置 │ 课次工作流 │ 多 AI 模块调度        │
└─┬──────────┬──────────┬──────────┬──────────┬────────────────┘
  ▼          ▼          ▼          ▼          ▼
听写技能   对齐技能   评课技能   结构技能   巩固技能
ASR+校对  翻页+PPT   评价建议   重难点+导图  课后习题
  │          │          │          │          │
  └──────────┴────┬─────┴──────────┴──────────┘
                  ▼
     SQLite 课堂记忆库 + 课件文件 + 用户知识库
```

### 3.3 核心功能模块（对照备课—授课—评价）

#### （1）备课：智能体工作台（可配置 Agent）

| 配置项 | 功能说明 | 教学价值 |
|--------|----------|----------|
| 角色设定 | 名称、任务目标、人设、能力边界 | 明确「依据证据分析、禁止臆造」等规则 |
| 评价标准 | 自定义评课文本与评分维度开关 | 对齐校级/专业评课要求 |
| 报告与作业 | 勾选报告章节；设置选择/填空/简答题量 | 决定课后输出形态 |
| 知识库 | 上传课标、讲义等（txt/md/pdf/pptx） | 为评课提供补充证据（**≠ 上课 PPT**） |
| 技能与工作流 | 听写/对齐/评课/结构/出题及流程步骤 | 固化感知—记忆—推理—行动剧本 |

#### （2）授课：课堂随行（感知 + 记忆 + 即时行动）

- **建课上课**：新建课次，上传上课 PPT，浏览器内放映与翻页；
- **开始听课**：启动实时语音转写，字幕上屏，服务学生边听边看；
- **翻页对齐**：同步记录页码与时间，形成语音—课件双通道记忆；
- **字幕增强**：ASR + LLM 术语校对 + 可选学科热词，提升专业课可用性。

#### （3）评价：成长报告（推理 + 行动）

结束听课后自动触发全量分析流水线，典型输出包括：

1. 教学评价与改进建议（综合分、分项分、亮点、建议）  
2. 课堂总结、重点、难点、知识点标签  
3. 思维导图（知识结构）  
4. 课后习题（按配置题型题量，依据本节讲授内容）  
5. 完整转写记录、课件回放、Word/Markdown 导出  

**证据约束**：分析以课件原文、课堂转写、知识库为准，禁止仅凭课程标题推断；评价与建议要求具体、可操作、可追溯。

### 3.4 智能体核心搭建步骤（可演示）

> 对应作品展示视频「核心搭建步骤」，建议按下列顺序录制。

| 步骤 | 搭建内容 | 对应能力 |
|------|----------|----------|
| 1 | 定义角色与能力边界 | 目标与行为约束 |
| 2 | 配置评价标准与维度 | 评课推理规则 |
| 3 | 配置报告项与作业题量 | 行动输出规格 |
| 4 | 上传知识库材料 | 长期知识记忆 |
| 5 | 启用技能与工作流 | 任务编排 |
| 6 | 建课上传上课 PPT | 授课感知准备 |
| 7 | 开始听课：字幕 + 翻页 | 感知—记忆闭环 |
| 8 | 结束并生成报告 / 回放导出 | 推理—行动闭环 |

### 3.5 技术实现要点（算法与系统）

| 评分关注点 | 本项目做法 |
|------------|------------|
| **算法先进性** | 实时语音大模型（ASR-LLM）+ 生成式大模型；字幕二次校对；多模块 Prompt 专能协同；课件文本解析与按页证据对齐 |
| **系统稳定性** | WebSocket 转写自动重连；重连后同步当前页码；课次状态机；账号鉴权与用户数据隔离 |
| **数据应用** | 合规采集课堂语音转写与翻页事件；结构化入库；仅用于本节课分析与复盘；配置与知识库按账号隔离；支持导出留存 |

**技术栈概要**：前端 React + Vite + TypeScript；后端 Express + WebSocket + SQLite；语音讯飞；推理 DeepSeek/通义千问；导出 docx。

## 四、应用成效及创新性

### 4.1 应用场景完整性（备课—授课—评价）

| 环节 | 教师/学生行为 | 智能体行为 | 成效 |
|------|----------------|------------|------|
| **备课** | 在工作台配置角色、标准、知识库、报告与题量 | 持久化 Agent 配置，后续分析自动读取 | 一次配置、多课复用；标准可校本化 |
| **授课** | 正常讲课、翻页 | 实时字幕、翻页时间轴入库 | 学生跟课更清楚；过程证据自动留存 |
| **评价** | 查看报告、回放、导出 | 自动生成评价/重难点/导图/习题 | 教研减负；反思有依据；复习资源一体化 |

### 4.2 应用成效（前后对照）

| 维度 | 应用前 | 应用后（智课随行） |
|------|--------|---------------------|
| 课堂记录 | 人工笔记，易遗漏 | 全程转写 + 按页归档 |
| 术语识别 | 通用 ASR 同音错误多 | AI 校对 + 可选专业热词 |
| 课后整理 | 约 30–60 分钟人工整理 | 结束听课后自动生成报告 |
| 教学评价 | 依赖主观印象，易套话 | 按自定义标准 + 课堂证据生成 |
| 复习资源 | 往往仅有课件 | 字幕回放 + 报告 + 习题一体化 |
| 多教师使用 | 配置与课次易混用 | 账号隔离，各自工作台与课次独立 |

**可验证效果表述（答辩可用）**：

- **教师侧**：将「听写整理—归纳重难点—写评课—出题」从课后连续劳动，压缩为「点结束并生成报告」一次触发；  
- **学生侧**：课中可看实时字幕，课后可看总结、导图与练习，降低跟课与复习门槛；  
- **教研侧**：Word 报告与同步回放形成可分享的数字课例，便于听评课与教学反思。

### 4.3 创新性（对照评分：功能创新 / 理念融合 / 场景适配）

#### （1）功能创新：突破「录音转写」或「课后聊天」的传统路径

提出并实现**课堂任务型智能体**：

1. **多模态课堂记忆**：语音转写 + PPT 翻页时间轴双通道，支持字幕与课件同步复盘；  
2. **「听—记—析—评」一体化流水线**：结束听课后自动完成总结、评价、导图、习题，而非多工具拼接；  
3. **可配置评课 Agent**：角色、标准、知识库、报告项、技能工作流可调，分析链路真实读取配置；  
4. **证据约束生成**：课件原文 × 口述转写 × 知识库联合推理，抑制「凭标题瞎编」。

#### （2）教育理念融合：对齐核心素养与过程性评价

- **以学习者为中心**：课中字幕提升信息可及性，体现差异化支持；  
- **过程性评价**：评价基于真实课堂证据，强调亮点与可操作改进，服务教师专业成长；  
- **教—学—评一致性**：报告与习题来自本节实际讲授内容，避免「教考脱节」；  
- **职业教育适切**：支持专业术语热词、技能课课件与理实讲解并存的课堂形态。

#### （3）场景适配：为中职真实课堂定制

- 交互按「开始听课 / 暂停 / 结束并生成报告」设计，贴合授课节奏；  
- 明确区分**上课 PPT（放映）**与**知识库（评课证据）**，避免概念混淆；  
- 低迁移成本：不要求教师改用新备课软件或全程对话交互；  
- 可推广：Web 部署 + 账号隔离，可在不同专业、不同学校复用同一智能体框架，仅替换标准与知识库。

### 4.4 用户体验与可推广性

| 维度 | 设计要点 |
|------|----------|
| 界面交互 | 功能导向工作台与课堂页；主流程短路径；品牌统一「智课随行」 |
| 响应效率 | 课中优先低延迟听写上屏；课后异步完成重分析任务 |
| 可推广性 | 不绑定单一学科；配置化标准与知识库即可迁移到酒店服务、信息技术等中职专业课，以及普通中小学学科课 |

### 4.5 数据合规与安全说明

- 采集范围限定为教学过程所需的语音转写、翻页事件、课件与知识库材料；  
- 用途限定于本节课字幕呈现、报告生成与复盘导出；  
- 按用户账号隔离配置、知识库与课次，降低混用与越权风险；  
- 建议在校园网/授权环境下使用，并遵循学校数据安全管理要求。

## 五、与「普通教学软件」的区别（创新性补充表述）

| 维度 | 普通教学/转写软件 | 智课随行（教育智能体） |
|------|-------------------|------------------------|

| 行动 | 文本结果 | 字幕、报告、回放、导出等工具化行动 |
| 个性化 | 难改评课逻辑 | 工作台配置驱动分析行为 |

## 六、技术文档摘要（架构与算法）

### 6.1 关键数据记忆

```
用户（鉴权）
 └── 智能体配置 / 知识库（按用户目录）
 └── 课次 sessions
      └── 课节 rounds
           ├── transcript_segments（转写记忆）
           ├── slide_events（翻页记忆）
           ├── analysis_results（教研+评价记忆）
           └── generated_questions（习题记忆）
```

### 6.2 推理编排入口

结束听课或手动「生成报告」时，调用全量分析编排：组装课件按页证据 + 完整转写 + 用户知识库与评价标准 → 依次/协同完成总结、评价、导图、出题 → 写入记忆并在报告页呈现。

### 6.3 核心目录索引

| 层级 | 路径 |
|------|------|

| 分析编排 | `server/ai/analyze.js` |
| 语音桥接 | `server/asr/` |
| 数据存储 | `server/db/` |

## 结语

「智课随行」以任务型教育智能体为形态，把中职课堂中「听不清、记不住、评不准、复盘难」的痛点，转化为可配置、可运行、可验证的智能体闭环：**课中字幕服务学，课后报告服务教**。其价值不在于堆叠更多按钮，而在于让教师保持原有教学习惯的同时，获得可沉淀的过程证据与可成长的评课支持。
